AngularJS的基本使用

By LanZhiSheng

发表于 2016-10-11

重要特性

MVC设计思想,model,controller和view
模块化,依赖注入
双向数据绑定

版本

1.X,我主要是自学了1.x,因为1.X用的最多2.X,4.X

指令

A.概念

AngularJS自定义的HTML属性标签

B.为什么会有

html在构建应用存在很多不足

C.具体分类

ng-app:指定作用范围
ng-show,hide:是否显示,隐藏,元素本身存在
ng-if:是否创建
ng-src,href:引入,引用
ng-class:添加类
ng-include=’path’,导入外部文件
ng-repeat=’item in arr’或ng-repeat=’(key,value) in arr’
ng-switch=’type’和ng-switch-when=’’配合使用

D.自定义指令

由于ng提供的指令不足以满足需求,使用directive方法

app.directive("hmTag", function() {
return {
    restrict: "A",
    //是否替换原标签
    replace: true,
    //模板
    template: "<h1>我是1个粉刷匠</h1>"
    //模板路径
    templateUrl:'./inlcude演示/top.html'
    //link可以对已经指令操作
    link: function(scope, element, attrs) {
        //
        element.on("click", function() {
            console.log("我被点了吗?");
        });
    }
}
});

单向和双向数据绑定

A.区别

单向数据绑定数据只能从控制器到视图,不可逆
双向是可逆的
绑定是ng-bind,双花括号是简写,

B.单向数据绑定
<body ng-app="hmApp">
<div ng-controller="demoCtrl">
    <p ng-cloak>{{user.name}}</p>
    <p ng-cloak>{{user.info}}</p>
</div>

<script>
    var app = angular.module("hmApp", []);
    app.controller("demoCtrl", ["$scope", 
        function($scope) {
        $scope.user = {
            name: "杰克.王",
            info: "跟着我"
        };
    }]);
</script>
</body>
C.双向实现方法

只作用于表单元素,设置ng-model指令

D.双向数据绑定原理

双向绑定的元素有一个watcher
事件触发(包括表单元素内容变化,ajax请求,点击按钮执行函数)调用digest脏数据检查
digest会检测所有的rootscope下所有被watcher元素

事件处理

原生的事件是全局变量,会造成全局变量污染

作用域

MVC的视图之间是可以相互嵌套的，存在父子关系，在子视图可以访问父视图的值
没有嵌套关系的作用域是相互独立的

过滤器

A.作用

在视图显示数据的时候，将要显示的数据进行一个处理之后再显示
在controller里面过滤,视图显示

B.分类
日期data:{{now|date:"yyyy-MM-dd HH:mm:ss"}}
货币currency:{{price|currency:"￥":1}}
转换小写lowercase:{{msg|lowercase}}
过滤器limitTo:{{data|limitTo:2}}
排序orderBy:{{stus|orderBy:'age':true}}

<span>时间{{now|date:"yyyy-MM-dd HH:mm:ss"}}
</span><br>
<span>价格是{{price|currency:"￥":1}}
</span><br>
<span>{{msg|lowercase}}</span><br>
<span>{{msg1|uppercase}}</span><br>
<span>{{data|limitTo:2}}</span><br>

app.controller("demoCtrl",["$scope",function
($scope){
    $scope.now = new Date();
    $scope.price = 113.45;
    $scope.msg = "THIS IS ITCAST!";
    $scope.msg1 = "hello,everyone!";
    $scope.data = [10,120,30,240,150,60,
    70,380,390,1100];
}]);

服务

A.概念

在ng中提供某一项功能

B.服务的使用

先在控制器参数里面依赖
然后在回调函数里面使用

C.分类

$scope是负责存贮数据模型
$log在控制台输出信息
$timeout/$interval定时服务

$filter过滤器服务,与filter过滤器的区别:filter过滤器用在视图中,在显示的数据模型数据过滤再显示,$filter是直接在控制器显示,直接改变数据模型

$http请求ajax,$http

app.controller("demoCtrl",["$scope",
 "$http", function($scope, $http) {
$http({
    url: "./php/stus.php",
    method: "post"
}).then(function(response) {
    console.log(response.data);
}, function() {
    console.log("请求失败!");
})
}]);
D.自定义服务

方法一:factory(),参数一自定义服务的名字,参数二:数组前面的元素要依赖的服务名,最后一个是回调函数

app.factory("hmDate", ["$filter", 
function($filter) {
return {
    now: function() {
        return $filter("date")(new Date,
 "yyyy-MM-dd HH:mm:ss");
    },
    format: function(format) {
     return $filter("date")(new Date, format);
    }
};
}]);

方法二:service参数1: 自定义服务的名称.参数2: 数组, 前面的元素写上要依赖的服务名称 形参注入

app.service("itcast",[function(){
     this.now = function(){
         return new Date();
        };
     this.sayHi = function(){
         console.log("hi.....");
        }
}]);

配置块

概念

ng在执行阶段,其中有一步骤.叫做配置和运行.这一阶段允许我们写代码来对ng进行一些配置.

方法

模块对象提供了一个方法叫做config。这个方法就允许我们对内置的服务进行配置.参数: 数组. 数组前面写上你要配置的服务对应的Provider最后1个元素是1个回调函数. 将Provider注入进行配置.

配置对应的provider
$log – $logProvider
$filter – $filterProvider
$http – $httpProvider

app.config(["$filterProvider", function
($filterProvider) {
$filterProvider.register("firstBig",
 function() {
    return function(data) {
        return data[0].toUpperCase() + 
data.slice(1);
    }
});
}]);

运行块run()

使用场景

Ng提供了很多的服务,$log,$http,$interval,$timeout,但是我们必须得将其注入到控制器中使用,2.这些服务其实可以脱离控制器运行,模块对象提供了run方法,可以直接允许ng服务

使用方法

参数:数组,里面有服务名称,最后一个元素回调

app.run(["$log","$http","$scope",
function($log,$http,$scope){
   $log.warn("这是一条警告消息");
   $scope.name = "jack";
 }]);

总结

基本使用主要包括:
ng指令:主要扩充html标签
过滤器:在view里对数据进行处理
服务:在controller里配合服务进行CRUD
配置块:可以通过服务的provider服务对该服务进行配置
运行块:之前服务在controller才可以用,但是有了run()也可以使用该方法
