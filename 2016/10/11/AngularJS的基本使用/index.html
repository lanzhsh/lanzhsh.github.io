AngularJS的基本使用

By LanZhiSheng

发表于 2016-10-11

重要特性

MVC设计思想,model,controller和view
模块化,依赖注入
双向数据绑定

版本

1.X,我主要是自学了1.x,因为1.X用的最多2.X,4.X

指令

A.概念

AngularJS自定义的HTML属性标签

B.为什么会有

html在构建应用存在很多不足

C.具体分类

ng-app:指定作用范围
ng-show,hide:是否显示,隐藏,元素本身存在
ng-if:是否创建
ng-src,href:引入,引用
ng-class:添加类
ng-include=’path’,导入外部文件
ng-repeat=’item in arr’或ng-repeat=’(key,value) in arr’
ng-switch=’type’和ng-switch-when=’’配合使用

D.自定义指令

由于ng提供的指令不足以满足需求,使用directive方法

app.directive("hmTag", function() {
return {
    restrict: "A",
    //是否替换原标签
    replace: true,
    //模板
    template: "<h1>我是1个粉刷匠</h1>"
    //模板路径
    templateUrl:'./inlcude演示/top.html'
    //link可以对已经指令操作
    link: function(scope, element, attrs) {
        //
        element.on("click", function() {
            console.log("我被点了吗?");
        });
    }
}
});

单向和双向数据绑定

A.区别

单向数据绑定数据只能从控制器到视图,不可逆
双向是可逆的
绑定是ng-bind,双花括号是简写,

B.单向数据绑定
<body ng-app="hmApp">
<div ng-controller="demoCtrl">
    <p ng-cloak>{{user.name}}</p>
    <p ng-cloak>{{user.info}}</p>
</div>

<script>
    var app = angular.module("hmApp", []);
    app.controller("demoCtrl", ["$scope", 
        function($scope) {
        $scope.user = {
            name: "杰克.王",
            info: "跟着我"
        };
    }]);
</script>
</body>
C.双向实现方法

只作用于表单元素,设置ng-model指令

D.双向数据绑定原理

双向绑定的元素有一个watcher
事件触发(包括表单元素内容变化,ajax请求,点击按钮执行函数)调用digest脏数据检查
digest会检测所有的rootscope下所有被watcher元素

事件处理

原生的事件是全局变量,会造成全局变量污染

作用域

MVC的视图之间是可以相互嵌套的，存在父子关系，在子视图可以访问父视图的值
没有嵌套关系的作用域是相互独立的

过滤器

A.作用

在视图显示数据的时候，将要显示的数据进行一个处理之后再显示
在controller里面过滤,视图显示

B.分类
日期data:{{now|date:"yyyy-MM-dd HH:mm:ss"}}
货币currency:{{price|currency:"￥":1}}
转换小写lowercase:{{msg|lowercase}}
过滤器limitTo:{{data|limitTo:2}}
排序orderBy:{{stus|orderBy:'age':true}}

<span>时间{{now|date:"yyyy-MM-dd HH:mm:ss"}}
</span><br>
<span>价格是{{price|currency:"￥":1}}
</span><br>
<span>{{msg|lowercase}}</span><br>
<span>{{msg1|uppercase}}</span><br>
<span>{{data|limitTo:2}}</span><br>
