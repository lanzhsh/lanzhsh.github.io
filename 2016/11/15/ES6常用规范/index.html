ES6常用规范

By LanZhiSheng

发表于 2016-11-15

兼容性

原因

由于ES6.1是2016年6月发布的,所以在此之前的浏览器都不支持,但是可以利用babel插件将ES6转化为ES5语法

解决方案

npm i babel-core
导入核心的browser.js

let和const

let:定义变量,没有预解析,有块级作用域,花括号可以分割作用域
const:定义常量,没有预解析,有块级作用域

解构赋值

数组和对象可以直接对应赋值

let [a,b,c]=[1,2,3]
let [x, y, z] = new Set([‘a’, ‘b’, ‘c’]);

数值扩展

0b(B)二进制
0o(O)八进制

箭头函数

var f = () => 5;
// 等同于
var f = function () { return 5 };

箭头函数返回的是一个对象,要用花括号
var sum = (num1, num2) => num1 + num2;
// 等同于
var sum = function(num1, num2) {
return num1 + num2;
};

对象的扩展

键值相同,值可以省略

module.exports = { getItem, setItem, clear };
// 等同于
module.exports = {
getItem: getItem,
setItem: setItem,
clear: clear
};

JS模块化

背景

1.css的模块化@import,最好不要用因为相对于link来说,@import是先加载html在加载css,link是遇到先加载完成;

2.js后端有commonJS,前端seaJS和requireJS

import输入

js引擎遇到import只会生成一个只读应用,脚本执行再去找值,原始值变加载值也变

import { stat, exists, readFile } from ‘fs’;

export输出

是编译时输出接口,commonJS是运行时加载

export function area(radius) {
return Math.PI radius radius;
}

import和export用法结合

需要明确知道要加载和输入变量名和函数名

export default默认输出

配合import解决模块文件输入输出

// export-default.js
export default function () {
  console.log('foo');
}

// import-default.js
import customName from './export-default';
customName(); // 'foo'

严格模式

1.ES6的模块采用严格模式

变量必须声明再使用
函数参数不能有同名属性
不能用with
this不指向全局
eval执行字符串代码时,作用域独立
