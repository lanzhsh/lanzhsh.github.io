闭包的应用及优缺点

By LanZhiSheng

发表于 2016-04-25

概念

闭包就是在函数里面使用局部变量

两种模型

A.函数模型
function getName(){
    var name='张三';
    function getName1(){
        console.log(name);
    }
}

B.对象模型
function getName(){
    var name='张三';
   return{ getName1:function getName1(){
        console.log(name);
        }
    }
}

应用

A.沙箱模式:
  1.运用自执行函数分割作用域,并用return暴露接口,
  可以解决全局变量污染问题;
  2.代码
  var name= (function(){
    var name='张三';
   return{ getName1:function(){
        console.log(name);
        }
    }
  })()

B.计数器:
var count1=(function(){
    var i=0;
    return {
        count: function(){
            console.log(i++);
        }
    }
})();
console.log(count1);
count1.count();
count1.count();

C.面向对象封装一些组件时缓存this

优缺点

A.优点:
可以访问函数外面的局部变量;
B.缺点:
会造成内存泄漏,所以要少用;

https,SSH和MD5加密原理与比较

By LanZhiSheng

发表于 2016-02-18

https

1.原理
A.就是在http加入SSL层,是http安全的基础;
B.htts协议是在http基础上加了SSL协议;
C.使用443端口,http是80端口;
D.由网景公司开发并内置在浏览器中;
2.作用
A.建立一个信息安全通道,来保证数据传输的安全;
B.确认网站的真实性,可以查看网站认证的真实性;
3.核心是SSL层

A.位置:是一个安全套层,是介于TCP?IP之间;
B.分类:
SSL记录协议:建立在TCP之上,为数据提供数据封装、压缩、加密;
SSL握手协议:建立在记录之上,数据传输之前双方的身份验证、协商加密算法、交换密钥;
C.SSL服务器认证过程:
客户端发送请求
服务器判断是否需要生成新主密钥,需要则返回
客户端收到信息,产生一个主密钥,并用公钥加密传输
服务器恢复该主密钥,并返回一个认证信息

4.SSL协议

A.SSL协议包括:握手(凭证交换和验证)和记录协议(数据进行加密);
B.SSL握手协议过程:
1.客服端发送SSL的版本号,算法种类,产生的随机数等;
2.服务器发送服务器的SSL协议的版本号,算法种类和证书(里面有公钥);
3.客服端验证;
4.验证通过随机产生对称密码,公钥加密产生预主密码,传输给服务器;
5.服务器可以要求验证身份,包括客户证书和签名随机数;
6.验证通过:服务器用私钥解开密码,对称密钥产生;
7.客服端发信息确认后面数据采用该通讯;
8.服务器端回应
C.整个过程涉及到3次请求,3次响应

SSH

1.概念

非对称性加密,包括传输层,用户认证和连接协议

2.原理

A,在a计算机中产生一对公钥和私钥
B,私钥不动,把公钥发送给计算机b,公钥很复杂,就算过程被窃取,第三方不懂什么意思
C,a和b首次进行尝试通讯,a发送的内容通过a存放的私钥进行加密,在网络中进行密文件传输
D,b也会对该端内容使用公钥解密,如果能两台计算机就结成亲密关系
E,以后a和b发送消息用私钥和公钥进行数据加密解密操作

3.用法

A.本地生成密钥:ssh-keygen -t rsa
B.密钥生成再c盘,将公钥方法github中,然后下次通讯就可以用SSH通讯

MD5

1.概念

信息摘要算法

2.特性

A.MD5加密输出是32位字符串;
B.相同的内容使用MD5加密后,得到的内容一样的;
C.MD5无法反向解密,唯一就是暴力碰撞破解(就是试32位密码);
D.为了防止暴力破解,可以加盐处理;

3.使用方法:

A.下载 npm i -S blueimp-md5;
B.引包
C.加密方式:
  普通加密:var val=md5('value');
  加盐:var val=md5('value','这是第一次加盐处理');

SSL,SSH和md5的对比

1.SSL是一个独立的安全套层加密协议;
  位于TCP(应用层)和IP(网络层)之间;
  只是起到将数据加密和验证作用;
2.SSH有加密,连接和认证功能;
3.md5只是简单讲数据加密传输或者被存贮到数据库;

本地存贮的三种方式

1.cookie
A.特性:

存贮大小只有4K,如果不设置存贮周期就是在浏览器会话期间;

B.原生设置:
//第一种：写入2个cookie
//document.cookie="name=张三";
//document.cookie="age=20";

//第二种方式：设置cookie有效期
//document.cookie="userAge=18;expires="+new Data
("2016-1-7 08:00:00");

//读取cookie：
console.log(document.cookie);
C.jquery设置:
 //1、设置会话cookie(关闭浏览器之后，数据就丢失了)
$.cookie("userName","c5");
$.cookie("userId","888");
//2、获取cookie值
console.log($.cookie("userName"));//"c5"

//3、设置cookie有效期
$.cookie("userAge","18",{
    expires:0.1   //7天
});

//4、删除cookie
$.removeCookie("userAge");
D.应用:

主要是存贮服务器返回的sessionId,并且下一次http请求传入服务器

2.localstorage和sessionstorage
A.特性:
前者20M,可以在多个窗口共享数据,周期永久,除非手动删除
后者5M,不能在多个窗口共享数据,周期为会话期间
两者都是h5出来的新对象
B.设置,获取与删除:
window.localstorage/sessionstorage.setItem()/
getItem()/removeItem();

localStorage.setItem('name',"张六");
 localStorage.setItem('age',22);
  console.log(localStorage.getItem('age'));
  console.log(localStorage.getItem('name'));
C.应用:
多窗口共享数据;存贮ajax请求拿到的数据,如购物车等
3.cookie和session解决的http无状态性问题
A.cookie

1.域名相同的cookie是通过请求头传递到服务器的,(浏览器的默认行为);
2.服务器的请求头可以通过req.header获取===>调用res处理===>cookie可以写入响应头.数据写入响应体,结束响应===>发送给客服端===>客服端可以自动拿到响应头和响应体;
3..cookie写入响应头方法

res.writeHeader(200, {
  'Content-Type': 'text/html; charset=utf-8',
  'Set-Cookie': ['isvisit=yes;'age=20']
});
B.session
1.为什么有了cookie还要用session?
  因为用户登录的信息贮存在浏览器端不安全.
2.原理:
  客户登录;
  服务器拿到数据验证,OK后登录成功;
  保存当前登录用户的状态和登录信息;
  服务器开辟内存空间贮存,并将sessionId返回cookie;
  第二次登录携带cookie的sessionId验证

离线缓存

1.概念

将互联网上的资源缓存到本地,断网也可以访问,主要是h5的application实现

2.实现方法(h5的新方法)
A.建一个manifest文件,并中引入
B.manifest文件主要内容:
CACHE MANIFEST
该符号后面的内容为注释信息，第一行不能少
CACHE:
需要缓存的文件
NETWORK:
 不需要缓存的文件
FALLBACK:
 当页面无法访问时的回退页面/重定向
C.js操作缓存:

缓存的属性:applicationCache.status(值为0-5);
缓存的事件:update(更新),updateready(已经更新完),progress(进度)

D.例子:

applicationCache.update();//更新

application.addEventListener(‘updateready’, function(){
console.log(‘已完成缓存的更新’);}, false);//已经完成

application.addEventListener(‘progress’, function(){
console.log(applicationCache.status); }, false);//进度

网络协议层之http协议

By LanZhiSheng

发表于 2015-11-16

网络协议层划分

OSI/RM,开放系统互连参考模型

7个分类

1.物理层:电力,光纤光缆
2.数据链路层:WiFi
3.网络层:IP(IPv4,IPv6),也就是localehost(本地主机),最常见的是webstorm和HBuilder编辑器
内置服务器打开编辑的项目就是该协议,实现原理是给回路网络接口一个标准主机名(相对的IP地址是127.0.0.1)
4.传输层:TCP
5.会话层:是建立在传输层之上，利用传输层提供的服务，使应用建立和维持会话
6.表示层:向上对应用层服务，向下接受来自会话层的服务。表示层为在应用过程之间传送的信息提供表示方法的服务
7.应用层:http和FTP

http协议

1.概念

A.超文本传输协议,是一个客户端和服务器端请求和应答的标准（TCP）。
B.在前端web发送请求主要使用http请求

2.http协议请求过程

A.用户输入通过URL;
B.三次握手:
  第一次客服端发送syn(同步序列编号)包到服务器;
  第二次服务器收到syn包,确认客户的SYN+ACK包,同时也发送一个SYN包;
  第三次客服端收到包,向服务器发送ACK确认包
C.三次握手TCP/IP连接成功;
D.DNS解析;
E.请求头(主要设置请求数据的编码格式和后台session返回的sessionID),请求行,请求主体。请求过程叫请求报文；
F.服务器处理并响应头,响应行,响应主体。这个过程叫响应报文。

3.http是无状态性的

就是说每个协议之间无关联，请求完成服务器完成忘记请求记录；
那么web页面的http请求怎么完成登录状态的保持?下次再总结。

ajax原理和封装

By LanZhiSheng

发表于 2015-10-29

概念

异步的JS和XML

XMLHttpRequest

1. 是ajax的核心机制;
2. 属性:readyState,请求状态,对应5个值;esponseText:服务器收到的响应体;responseXML:
返回XML;status:状态代码;statusText:属性名称指定状态代码;
3. 事件:Onreadystatechange,每次readyState属性改变调用的事件,因为load事件有兼容性
4. 方法:abort()取消响应,取消后readyState为0;setRequestHeader()设置HTTP请求;请求行open

实现原理

通过XMLHttpRequest对象的get/post发送请求头,请求行,请求主体,服务器响应头,响应行,响应主体

get和post发送请求

1.get

if (window.XMLHttpRequest) {  
var xhr = new XMLHttpRequest();  
} else {  
var xhr = new  ActiveXObject();  
}  
xhr.open('get', url ? 'jack=18' / xhr.open);
xhr.onload = function() { xhr.responseText };

2.post

if (window.XMLHttpRequest) {
    var xhr = new XMLHttpRequest();
} else {
    var xhr = new ActiveXObject();
}
xhr.open('post', url);
xhr.onload = function() { xhr.responseText };
xhr.send('jack=18');

3.get和post的区别:

A.get能在URL看到参数而post看不到;
B.get因为是拼接到url后面是有大小限制的,post如果服务器没有限制大小就可无限上传
C.url因为提交到服务器是拼接到url后面是QueryString类型,后台可以直接解析,
D.以post方式向服务器发送数据,默认数据格式是RequestPayload,这种格式在PHP中$POST无法获取(解析);
E.post还有一种格式Form Data,服务器可以正常拿到
